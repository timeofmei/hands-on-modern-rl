# 3.5 从 Q 到 Q-Learning

## 本节导读

**核心内容**

- **只有 V 表不够用**：没有环境模型时，知道局面好不好并不能帮你做决策。
- **把表扩展一维（Q 表）**：给“每个状态-动作对”单独存一个分数。
- **贝尔曼最优方程**：完美的 Q 表前后两步之间必须满足的递推关系。
- **Q-Learning**：走一步，算一下 TD Target，改一次表。
- **Off-policy 与探索**：一边允许自己随机试错，一边在心里学习最理智的走法。

在上一节，我们学会了用 DP、MC 和 TD 更新状态价值表 $V(s)$。有了这张表，我们知道了每一个状态“值多少钱”。但这还不够。

真正的问题在于：**如果不知道环境的规则，光知道状态好不好，并不能帮你做决定。**

先看直觉：假设你在迷宫里，面前有左、右两条路。你查了 $V$ 表，发现“拿到宝箱的房间”价值是 100 分，“掉进陷阱的房间”价值是 -100 分。但这毫无意义，因为你根本不知道“向左走”是通向宝箱还是陷阱。要想用 $V$ 表做决策，你必须有一个“环境说明书”（转移概率），告诉你每个动作会把你带去哪里。但在大多数真实的强化学习任务中，这本说明书是不存在的。

所以，我们要换个角度：既然不知道动作的后果，能不能直接把动作也评估了？不要问“这个房间有多好”，而是直接问“在这个房间向左走，有多好”？

这就是动作价值表，也就是 **Q 表**。

::: info 核心概念
Q-Learning 学的不是一条固定路线，而是一张**动作价值表**。它是状态价值表的进阶版：不再只给每个状态存一个 $V(s)$，而是给每个状态下的每一个动作单独存一个 $Q(s,a)$。
:::

**核心公式**

$$
Q^\pi(s,a)=\mathbb{E}_\pi[G_t\mid S_t=s,A_t=a]
\quad \text{（动作价值函数：先指定动作，再评价未来回报）}
$$

$$
Q^*(s,a) = \mathbb{E}\left[ r+\gamma\max_{a'}Q^*(s',a') \mid s,a \right]
\quad \text{（贝尔曼最优方程：定义最优动作价值的自洽关系）}
$$

$$
Q(s,a) \leftarrow Q(s,a) + \alpha\left[ r+\gamma\max_{a'}Q(s',a')-Q(s,a) \right]
\quad \text{（Q-Learning 更新：用一次交互经验修正动作价值）}
$$

> **本节公式的作用：**
>
> 第一行定义了 Q 表中每个格子应该代表什么；第二行说明了完美的 Q 表前后两步之间必须满足什么递推关系；第三行把这个递推关系变成了一个可以通过不断试错来“改表”的规则。理解这三行公式，就能看清 Q-Learning 的核心：**用采样到的一步经验，让整张 Q 表逐渐逼近贝尔曼最优方程的解。**

## 把表扩展一维：Q(s, a)

状态价值表 $V(s)$ 只有一列，它回答的是“局面整体好不好”。
动作价值表 $Q(s,a)$ 则多了一维：每一行是一个状态，每一列是一个动作。

| 状态 $s$ | 向上走 | 向右走 | 向下走 | 向左走 |
| -------- | ------ | ------ | ------ | ------ |
| $(0,0)$  | -5.1   | -4.6   | -4.6   | -5.2   |
| $(0,1)$  | -4.2   | -3.4   | -3.8   | -4.8   |

现在，站在同一个格子里，向右和向下可能都能到终点，但分数会有高低。有了这张表，决策就变得极其简单：查一下当前状态这一行，**哪个动作分数最高，就选哪个**。也就是：

$$
\pi^*(s) = \arg\max_a Q^*(s,a)
$$

完全不需要环境模型。

Q 表里每个格子的数学含义是这样的：

$$
Q^\pi(s,a) = \mathbb{E}_\pi [G_t \mid S_t=s, A_t=a]
$$

这句话的意思是：如果现在处在状态 $s$，**先强制执行动作 $a$**，从这以后都按既定策略 $\pi$ 继续行动，平均能拿多少回报。

真正的问题是：这张表一开始全是 0（或者随机数），智能体该怎么把它填准？

## 贝尔曼最优方程：完美 Q 表的模样

在写出学习算法之前，我们先问一个更基础的问题：如果 Q 表已经被填得非常完美（记作 $Q^*$），它应该长什么样？

先看直觉：你做了一个动作 $a$，拿到了眼前的奖励 $r$，并且进入了下一个状态 $s'$。到了 $s'$ 之后，既然你有一张完美的 Q 表，你肯定会查表并选择 $s'$ 里分数最高的那个动作。

所以，当前动作的真实价值，理应等于“眼前的奖励”加上“下一步最好动作的价值（打个折扣）”。

写成公式，就是**贝尔曼最优方程**：

$$
Q^*(s,a) = \mathbb{E}\left[ r + \gamma \max_{a'} Q^*(s',a') \mid s,a \right]
$$

这不是在凭空造公式，而是在描述一个事实：真正的最优 Q 表，它的前后两步之间，必须严丝合缝地满足这个递推关系。而 Q-Learning 要做的，就是通过不断试错，把一张全是 0 的空表，一点点修改成满足这个等式的完美表。

## Q-Learning：走一步，改一次表

怎么改？和上一节的 TD 方法一样，我们不等整局结束，**走一步就改一次**。

假设你现在在状态 $s$，做了动作 $a$，得到了奖励 $r$，来到了下一个状态 $s'$。你回头看一眼刚才的动作，心里算一笔账：

我之前以为这个动作值 $Q(s,a)$（旧预测）。
但我现在真实拿到了 $r$，而且我看到下一个状态 $s'$ 里最好的动作目前值 $\max_{a'} Q(s',a')$。

所以我现在认为，这个动作的目标分数应该是：

$$
\text{TD Target} = r + \gamma \max_{a'} Q(s',a')
$$

这叫 **Bootstrapping（自举）**：用自己对未来的估计，来更新当前的估计。听起来有点左脚踩右脚，但正是它让智能体不必等到游戏结束就能边走边学。

目标和旧预测之间的差距，就是 **TD Error（时序差分误差）**：

$$
\delta = \text{TD Target} - Q(s,a)
$$

最后，按学习率 $\alpha$ 把旧预测往新目标挪一点：

$$
Q(s,a) \leftarrow Q(s,a) + \alpha \delta
$$

合起来，这就是大名鼎鼎的 Q-Learning 更新公式：

$$
Q(s,a) \leftarrow Q(s,a) + \alpha\left[ r + \gamma \max_{a'} Q(s',a') - Q(s,a) \right]
$$

## 手算一次更新

我们用一个极简的场景来算一次。假设你在一个 4×4 的网格世界里，每走一步扣 1 分（鼓励尽快找到出口），折扣因子 $\gamma=0.9$。

假设你刚开始学，Q 表全是 0。你在起点 $(0,0)$ 向右走了一步，到了 $(0,1)$，系统立刻扣了你 1 分（$r=-1$）。

![Q-Learning 单步更新示意图](./images/q-learning-simple-update.svg)

1. **旧预测**：起点向右走的价值，查表发现 $Q((0,0), \text{右}) = 0$。
2. **看未来**：到了 $(0,1)$ 后，查表发现那里的四个动作目前的分数都是 0，所以 $\max_{a'} Q((0,1), a') = 0$。
3. **算目标**：$\text{TD Target} = -1 + 0.9 \times 0 = -1$。
4. **改表格**：假设学习率 $\alpha=0.1$。新 $Q = 0 + 0.1 \times (-1 - 0) = -0.1$。

这一步很小，但意义深远。智能体原本以为“起点向右走”是 0 分，现在它被“烫”了一下，把这个动作的分数调低到了 $-0.1$。如果它反复尝试，分数会越降越低，直到它发现了终点。随着无数次这种微小的修改，终点附近的“好分数”会像波纹一样，一步步倒退着传回起点。

## 用代码跑一个 GridWorld

我们把上面说的“走一步、算目标、改表格”翻译成极简的 Python 代码。不依赖任何强化学习库，只有最核心的逻辑。

```python
import numpy as np

rng = np.random.default_rng(0)

N = 4
START = (0, 0)
GOAL = (3, 3)
ACTIONS = [(-1, 0), (0, 1), (1, 0), (0, -1)]  # 上、右、下、左
ARROWS = np.array(["↑", "→", "↓", "←"])

def to_state(pos):
    return pos[0] * N + pos[1]

def to_pos(state):
    return divmod(state, N)

def step(state, action):
    row, col = to_pos(state)
    if (row, col) == GOAL:
        return state, 0, True

    dr, dc = ACTIONS[action]
    next_row = min(max(row + dr, 0), N - 1)
    next_col = min(max(col + dc, 0), N - 1)
    next_state = to_state((next_row, next_col))
    done = (next_row, next_col) == GOAL
    return next_state, -1, done

Q = np.zeros((N * N, len(ACTIONS)))
alpha = 0.1
gamma = 0.9
epsilon0 = 0.3

for episode in range(2000):
    state = to_state(START)
    # 探索率逐渐衰减
    epsilon = max(0.02, epsilon0 * (0.995**episode))

    for _ in range(100):
        # 1. 选动作（探索与利用）
        if rng.random() < epsilon:
            action = int(rng.integers(len(ACTIONS)))
        else:
            best_actions = np.flatnonzero(Q[state] == Q[state].max())
            action = int(rng.choice(best_actions))

        # 2. 和环境交互
        next_state, reward, done = step(state, action)
        
        # 3. 计算 TD Target 并更新 Q 表
        bootstrap = 0 if done else Q[next_state].max()
        target = reward + gamma * bootstrap
        Q[state, action] += alpha * (target - Q[state, action])

        state = next_state
        if done:
            break

print("起点四个动作的 Q 值：")
print(dict(zip(ARROWS, Q[to_state(START)].round(3))))

print("\n学到的一种贪婪策略：")
for row in range(N):
    cells = []
    for col in range(N):
        if (row, col) == GOAL:
            cells.append("G")
        else:
            state = to_state((row, col))
            cells.append(ARROWS[int(Q[state].argmax())])
    print(" ".join(cells))
```

运行代码，你会看到类似这样的输出：

```text
起点四个动作的 Q 值：
{'↑': -4.797, '→': -4.686, '↓': -4.686, '←': -4.919}

学到的一种贪婪策略：
→ ↓ → ↓
↓ → → ↓
→ → → ↓
→ → → G
```

仔细看起点向右和向下的 Q 值：$-4.686$。如果你还记得等比数列，最短路径需要 6 步，其折扣回报正好是：
$-1 - 0.9 - 0.9^2 - 0.9^3 - 0.9^4 - 0.9^5 = -4.68559$。

Q-Learning 没有在脑海中做任何宏观的“路线规划”。它只是盲目地在格子里走来走去，走一步改一次表，最终却拼凑出了全局的最优解。

## 探索与 Off-policy：一边乱走，一边学最优

细心的你可能发现了一个问题：如果一开始 Q 表全是 0，智能体怎么知道该往哪走？

如果总是选当前 Q 值最高的动作，它可能会在起点的某个角落里一直打转（撞墙）。没吃过苹果，就不会知道苹果甜。这就引出了**探索（Exploration）**。

在上面的代码里，我们用了 **$\varepsilon$-贪婪策略**：大部分时候选最高分的动作，但有 $\varepsilon$ 的概率随机乱走。一开始 $\varepsilon$ 很大，后来慢慢衰减。这就保证了所有的格子、所有的动作迟早都会被试到。

但这就带来了一个非常奇妙的性质：**行为和学习的分离**。

智能体在实际走路时，像个醉汉，偶尔会随机试探（这叫**行为策略**）。但是，它在更新 Q 表时，公式里写的是 $\max_{a'} Q(s',a')$。这意味着它在心里盘算的是：“虽然我刚刚因为探索瞎走了一步，但如果我未来变得完全理智，每次都选最优动作，能拿多少分？”（这叫**目标策略**）。

这种实际走的策略，和心里想学的策略不一致的算法，叫做 **Off-policy（异策略）**。Q-Learning 是一边允许自己犯错，一边在学习“如果以后不犯错，应该怎么走”。

## 悬崖边上的性格测试：Q-Learning 对比 SARSA

为了把 Off-policy 的影响看透，我们来看一个悬崖行走的例子（Cliff Walking）。

```text
.  .  .  .  .  .  .  .  .  .  .  .
.  .  .  .  .  .  .  .  .  .  .  .
.  .  .  .  .  .  .  .  .  .  .  .
S  C  C  C  C  C  C  C  C  C  C  G
```

起点是 S，终点是 G，中间 C 全是悬崖。掉进悬崖扣 100 分并回到起点，普通走一步扣 1 分。

从 S 到 G 最短的路线，显然是“贴着悬崖边走”。
- **Q-Learning** 因为心里总是假设未来会采取完美动作（使用 $\max$），所以它学出来的结果就是这条**贴边的最短路**。
- 但如果我们把 Q-Learning 公式里的 $\max$ 去掉，换成“下一步实际凭 $\varepsilon$-贪婪选出的动作”，算法就变成了 **SARSA**（On-policy）。

SARSA 的更新目标是 $r + \gamma Q(s', a_{\text{actual}})$。因为它知道自己当前还有 $\varepsilon$ 的概率会乱走，贴着悬崖走很容易一不小心就掉下去。所以，SARSA 最终学到的策略会是“绕远路”，宁可多扣几分，也要离悬崖远远的。

这不是谁更聪明的问题，而是它们回答的问题不同：
- Q-Learning 回答的是：“如果抛开我探索时的笨手笨脚，**理想最优**是什么？”
- SARSA 回答的是：“带着我目前的笨手笨脚，怎么走**最安全**？”

## 表格方法的边界

到这里，Q-Learning 似乎已经完美解决了小网格里的决策问题。不需要环境模型，不用等整局结束，每走一步就能更新。

但它有一个非常致命的前提：**状态和动作的数量，必须能被表格装下。**

4×4 GridWorld 只有 64 个 Q 值。可是一旦换成控制机器人，状态包含无数种角度和速度组合，是连续的无限个状态。再换成打游戏，状态是一帧帧图像，像素组合的数量比宇宙里的原子还多。表格根本存不下。

所以，Q-Learning 的核心更新思想没有过时，过时的是“每个状态-动作对都单独存一行”这件事。第 4 章要解决的正是这个问题：如果表格装不下，能不能用一个神经网络来“近似”整张 Q 表？答案就是深度 Q 网络（DQN）。

## 小结

- **动作价值函数 $Q(s,a)$** 可以脱离环境模型直接用于决策：只需选出 Q 值最大的动作。
- **Q-Learning 的核心**是用 TD 目标 $r+\gamma\max_{a'}Q(s',a')$ 去修正旧估计，让 Q 表逼近贝尔曼最优方程。
- **$\varepsilon$-贪婪**用于在“探索未知”和“利用已知”之间寻找平衡。
- **Off-policy**：Q-Learning 的实际行为策略（带探索）和学习目标策略（完全贪婪）是分离的。
- 状态空间一旦变大，表格方法就会失效，必须引入函数逼近（深度强化学习）。
